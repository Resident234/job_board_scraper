# Стратегия повторов для HTTP ошибок

## 🎯 Обзор

Система автоматических повторов HTTP-запросов реализована в `ProxyRetryExecutor` с использованием экспоненциального роста задержек.

## 🔍 Проблема

При работе с прокси часто возникают ошибки:
- **500 Internal Server Error** - внутренняя ошибка сервера
- **502 Bad Gateway** - прокси не может подключиться к серверу
- **503 Service Unavailable** - сервер временно недоступен
- **403 Forbidden** - IP заблокирован или доступ запрещён
- **429 Too Many Requests** - превышен лимит запросов (rate limiting)
- **408 Request Timeout** - таймаут запроса

**Важно:** Страница считается обработанной ТОЛЬКО при получении ответа 200 OK. Любой другой код ответа приводит к повторной попытке со сменой прокси.

## 🛠️ Решение

### Обрабатываемые коды ответов

| Код | Описание | Поведение |
|-----|----------|-----------|
| 200 | OK | Успех, сброс счётчика ошибок прокси |
| 404 | Not Found | Записывается "Ошибка 404" в title/about, без повторов |
| 5xx | Server errors | Повтор с экспоненциальной задержкой |
| 403 | Forbidden | Смена прокси, повтор |
| 429 | Rate limited | Retry-After или увеличенная задержка |
| 408 | Request timeout | Смена прокси, повтор |
| Другие | Любой не-200/404 | Смена прокси, повтор |

### Алгоритм задержек

**Для ошибок сервера (5xx):**
- Базовая задержка: 2000 мс
- Максимальная: 60000 мс
- Экспоненциальный рост: 2x на каждой попытке
- Jitter: 30%

**Для ошибок прокси/сети (403, 408, 502, 503):**
- Базовая задержка: 500 мс
- Максимальная: 10000 мс
- Jitter: 20%

## 🔧 Реализация

### ProxyRetryExecutor

**Класс:** `JobBoardScraper/Infrastructure/Proxy/ProxyRetryExecutor.cs`

Основные методы:
- `ExecuteAsync(...)` - выполнение запроса с повторами
- `ReportSuccessSafe(...)` - статический метод для отчёта об успехе
- `ReportDailyLimitSafe(...)` - статический метод для отчёта о суточном лимите
- `HandleDailyLimit(...)` - обработка суточного лимита

## 📊 Поведение

### При успешном ответе (200):
- Сбрасывается счётчик ошибок прокси
- При успешной обработке — прокси добавляется в whitelist

### При ошибке 404:
- Записывается "Ошибка 404" в поля title и about
- Профиль помечается как обработанный
- Без повторных попыток

### При других ошибках:
- Вычисляется задержка
- При наличии прокси — повтор через другой прокси
- При отсутствии прокси — прямой повтор
- При превышении `MaxSwitches` — прекращение попыток

## 🔄 Алгоритм работы

1. `ProxyHttpClientFactory` отправляет запрос
2. При получении кода ответа:
   - **200 OK** → успех, `ReportSuccessSafe()`
   - **404** → фиксация как "обработано"
   - **Другие** → `ProxyRetryExecutor.ExecuteAsync()`
3. При повторе:
   - Рассчитывается задержка
   - Ждёт, затем повторяет запрос
   - При превышении лимита ошибок — прокси удаляется
