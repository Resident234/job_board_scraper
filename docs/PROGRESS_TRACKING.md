# 📊 Потокобезопасное отслеживание прогресса (Progress Tracking)

Этот документ описывает систему отслеживания прогресса скрапинга в проекте Job Board Scraper.

---

## 🎯 Обзор

В процессе скрапинга больших объемов данных (тысячи профилей пользователей, сотни компаний) важно отслеживать процент выполнения работы в режиме реального времени. Поскольку задачи выполняются параллельно и асинхронно в нескольких потоках, стандартные инкрементные счетчики не подходят из-за проблем с потокобезопасностью (состояние гонки / race conditions).

Для решения этой задачи в проекте реализован класс `ProgressTracker`, который обеспечивает атомарное, потокобезопасное обновление и чтение прогресса выполнения.

---

## 🛠️ Ключевые компоненты

### 1. [ProgressTracker](file:///h:/s/Work_job_board_scraper/job_board_scraper/JobBoardScraper/Infrastructure/Utils/ProgressTracker.cs)

Служебный класс, инкапсулирующий состояние прогресса:
- **`Processed`** (обработано задач) — инкрементируется атомарно.
- **`Total`** (всего задач) — общее число запланированных к обходу элементов.
- **`Percent`** — вычисляемое свойство текущего прогресса в процентах.
- **`IsComplete`** — флаг завершения работы (`Processed >= Total`).
- **`TaskName`** — строковое название текущей задачи (например, `UserProfiles` или `CompanyDetails`).

Для обеспечения многопоточности все изменения выполняются с помощью класса `System.Threading.Interlocked`:

```csharp
// Атомарный инкремент счетчика обработанных задач
public int Increment() => Interlocked.Increment(ref _processed);

// Безопасный сброс параметров
public void Reset(int newTotal)
{
    Interlocked.Exchange(ref _processed, 0);
    Interlocked.Exchange(ref _total, newTotal);
}
```

### 2. [ScraperProgressLogger](file:///h:/s/Work_job_board_scraper/job_board_scraper/JobBoardScraper/Infrastructure/Logging/ScraperProgressLogger.cs)

Высокоуровневый фасад, объединяющий `ProgressTracker` и `ConsoleLogger`. Упрощает логирование типовых событий с автоматическим выводом текущего прогресса:
- `LogHttpProgress` — логирует завершение HTTP-запроса, время выполнения, статус-код, состояние прогресса и количество параллельно выполняемых запросов.
- `LogPageProgress` — логирует обработку пагинационных страниц со списками компаний или резюме.
- `LogItemProgress` — логирует обработку единичного элемента.

## 🔄 Использование в скраперах

Класс `ProgressTracker` интегрирован во все основные скраперы проекта через `ScraperProgressLogger`:

1. **Инициализация**:
   При запуске процесса скрапинга вычисляется общее количество ссылок или записей к обработке, и создается логгер прогресса:
   ```csharp
   _progress = new ProgressTracker(totalLinks, "UserProfiles");
   ```

2. **Инкремент прогресса**:
   После обработки каждого URL (успешно, с ошибкой или пропущено) вызывается метод инкремента счетчика:
   ```csharp
   _progress.Increment();
   ```

3. **Вывод в лог**:
   Логирование прогресса автоматически выводит данные в формате:
   `[TaskName] Processed/Total (Percent%)`
   
   *Пример вывода:*
   ```text
   [UserProfiles] 150/1000 (15.0%)
   ```

---

## 💡 Преимущества подхода

1. **Thread-Safety (Потокобезопасность)**: Полное отсутствие блокировок (`lock`) за счет использования быстрых атомарных инструкций процессора (`Interlocked`).
2. **Точность**: Счетчик никогда не потеряет инкремент из-за конфликта потоков при высокой степени параллелизма.
3. **Читаемость логов**: Единый структурированный формат отображения прогресса по всем модулям скрапинга.
